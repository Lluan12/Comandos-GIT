Glosario de Comandos de GIT

Comando Descripción Parametros Ejemplo
Configuración
git config −−global user.name “[nombre-usuario]” Establecer un nombre que sea identificable para cuando se revise el historial de versiones [nombre-usuario] git config −−global user.name "lluan12"
git config −−global user.email “[email válido]” Establecer una dirección de correo electrónico que se asociará a cada commit del historial [email válido] git config −−global user.email "isllhernandezgo@ittepic.edu.mx"
Inicialización
git init Inicializa un repositorio GIT en el directorio actual SP git init
git clone [url] Recuperar un repositorio completo desde una ubicación alojada a través de una URL [url] git clone https://github.com/Lluan12/Comandos-GIT.git
Escenario
git status Muestra los archivos modificados en el directorio de trabajo, preparados para su próximo commit SP git status
git add [fichero] Añadir un archivo para el próximo commit [fichero] git add README.md
git add -u Actualiza el índice sólo donde ya tiene una entrada que coincide con "pathpec" SP git add -u
git reset [fichero] Desenvolver un archivo conservando los cambios en el directorio de trabajo [fichero] git reset hola.txt
git reset −− soft HEADˆ Desenvolver el último commit conservando los cambios en el directorio local SP git reset −− soft HEADˆ
git diff Diferencia de lo que se cambia pero no se pone en escena SP git diff
git diff −−staged Diferencia de lo que se ha puesto en escena pero aún no se ha hecho commit SP git diff −−staged
git commit -m “[mensaje descriptivo]” Hacer un nuevo commit con su respecta descripción [mensaje descriptivo] git commit -m “Primer commit”
Ramas
git branch Listar las ramas y aparecerá un * al lado de la rama actualmente activa SP git branch
git branch [nombre de la rama] Crear una nueva rama en el commit actual [nombre de la rama] git branch desarrollador1
git checkout -b [nombre de la rama] Creación y desplazamiento a la rama indicada [nombre de la rama] git checkout -b desarrollador2
git branch -d [nombre de la rama] Elimina la rama indicada [nombre de la rama] git branch -d desarrollador1
git checkout [nombre de la rama] Cambiar a otra rama y comprobarla en el directorio de trabajo [nombre de la rama] git checkout main
git checkout −− . Retomar el estado del último commit SP git checkout −− .
git merge [rama] Fusionar el historial de la rama especificada en la actual [rama]: Nombre de la rama git merge desarrollador2
Inspeccionar y Comparar
git log Mostrar el historial de commits de la rama actualmente activa SP git log
git log [ramaB...ramaA] Mostrar los commits de la ramaA que no están en la rama B [ramaB...ramaA]: Nombre de la rama A y B git log desarrollador1...desarrollador2
git log −−follow [file] Mostrar los commits que cambiaron el archivo, incluso a través de los renombramientos [file] git log −−follow calculo.py
git diff ramaB...ramaA Mostrar la diferencia de lo que hay en la rama A que no está en la rama B [ramaB...ramaA]: Nombre de la rama A y B git diff desarrollador1...desarrollador2
Seguimiento de los cambios
git rm [fichero] Eliminar el archivo del proyecto y preparar la eliminación para el commit [fichero] git rm README.md
git mv [ruta existente] [nueva ruta] Renombrar un archivo o directorio o cambiar la ruta de un archivo existente y realizar el movimiento [ruta existente]: Ruta actual [nueva ruta]: Nueva ruta a mover git mv hola.txt script/hola.txt
git log −−stat -M Mostrar todos los registros de commit con indicación de las rutas que se han movido SP git log −−stat -M
Compartir y Actualizar
git remote add [alias] [url] Añadir una URL de git como alias [alias]: Nombre de variable [url]: Direccin del repositorio git remota add origin https://github.com/Lluan12/Comandos-GIT.git
git fetch [alias] Obtener todas las ramas de ese Git remoto [alias]: Nombre de variable git fetch origin
git merge [alias]/[rama] Fusionar una rama remota en su rama actual para actualizarla [alias]: Nombre de variable [rama]: Nombre de la rama git merge origin prueba
git push [alias] [rama] Transmitir los commits de la rama local a la rama del repositorio remoto [alias]: Nombre de variable [rama]: Nombre de la rama git push origin prueba
git pull Obtener y fusionar los commits de la rama remota a la rama local SP git pull
Tags
git tag Visualizar todos los tags SP git tag
git tag -a [ID del tag] -m "[mensaje descriptivo]" Crear un tag para el último commit realizado con un mensaje enlazado a este tag [ID del tag]: Nombre para reconocer el tag [mensaje descriptivo] git tag -a v1.0.0 -m "Primer version completa"
git tag -a [ID del tag] [hash del commit] -m "[mensaje descriptivo]" Crear un tag para un commit identificado con su hash [ID del tag]: Nombre para reconocer el tag [hash del commit]: hash para reconocer el commit [mensaje descriptivo] git tag -a v2.0.0 afcer3 -m "Seguna version completa"
git tag -d [ID del tag] Eliminar el tag específicado [ID del tag]: Nombre para reconocer el tag git tag -d v1.0.2
git push −−tags Enviar todos los tags al repositorio remoto SP git push --tags
Reescribir la historia
git rebase [rama] Aplicar cualquier commit de la rama actual por delante de la especificada [rama]: Nombre de la rama git rebase dev
git reset −−hard [commit] Despejar el escenario y reescribir el árbol de trabajo a partir del commit específicado [commit]: Hash del commit a reescribir git reset --hard adfad3
Commits Temporales
git stash Almacenamiento de los cambios realizados en la "pila de almacenamiento" SP git stash
git stash list Listar el orden de pila de los cambios en los archivos almacenados SP git stash list
git stash pop Toma un cambio almacenado, lo elimina de la "pila de almacenamiento" y lo aplica al árbol de trabajo actual SP git stash pop
git stash drop Descartar los cambios de la parte superior de la "pila de almacenamiento" SP git stash drop
git stash clear Borrar todas las entradas del stash list SP git stash clear
git stash apply Restaurar el último registro del stash SP git stash apply
git stash apply [ID del stash] Restaurar el stash identificado con su ID [ID del stash]: Valor mostrado en el stash list git stash apply drte4
git stash list −−stat Aparece más información para cada uno de los stash de los registros SP git stash list −−stat
git stash show Información detallada del stash SP git stash show
